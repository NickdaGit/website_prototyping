# MonkeyMagic.ai Project Context & Setup for Agentic AI

Hey—you're helping me build **monkeymagic.ai**. This is your briefing: who owns it, what it should feel like, how it works, and why. Read this first—then go write, chunk, link, or style whatever I ask.

## Who & What
- Owner: Nicholas (that's me).  
- Domain: monkeymagic.ai — pseudo-company, not real LLC, just fun branding.  
- Hardware: Mac Mini, VS Code, GitHub Copilot Pro (so I can code fast, but still want you doing the heavy lifting).  
- Goal: A knowledge base that's useful, layered (beginner → expert), agent-ready (RAG, vectors, local Chroma), but wrapped in a cheeky, lighthearted monkey theme. Think "smart library meets circus"—no corporate stiffness.

## Vibe & Branding
- Fun first: Logo = cartoon monkey waving a wand (Monkey Magic!).  
- Hero: Video of an ape strolling up, sitting at desk, typing code—like "even primates code now".  
- Tone: Light, jokey, warm. "Where monkeys code, dream, and occasionally steal bananas."  
- Navigation: Top bar—About | Knowledge | Fun Stuff. Clean, bold, playful.  
- First impression: Feels like a quirky startup site—logo top-left, hero banner, then nav. No dark academia, no sterile docs.

## Sections & Taglines
- **Knowledge**: "Bold Knowledge" button → tagline "For Curious Little Monkeys" (leads to the layered MD repo).  
- **Fun Stuff**: "For Cheeky Little Monkeys" — silly bits, memes, banana-code jokes, maybe a "monkey tricks" tutorial.  
- **About**: "We're not a real company—just monkeys with keyboards." Goes to Team page with monkey avatars + bios:  
  - **Gunther** (lead dev): capuchin, codes at banana speed, ethics "mostly", loves regex.  
  - **Lila** (design queen): gorilla, anti-clutter except peels, sunglasses on point.  
  - **Zara** (data wizard): bonobo, predicts banana shortages, trains models on fruit prices.  
  All with little backstories—keep 'em short, funny, no cringe.

## Knowledge Section (The Real Meat)
- Lives at `/knowledge/` — all our MD files.  
- Human view: index.html with collapsible folders (AI, Science, etc.), difficulty badges, raw .md links.  
- Agent view: Same files—YAML metadata, small chunks, local embeddings.  
- Style: Friendly tutor voice—short, clear, active, no hype. See writing-style-guide.md.  

## Design Rules
- Simple but effective: HTML + CSS only—no frameworks, no npm hell.  
- Works everywhere: phone, desktop, old browsers, offline.  
- Looks nice: system fonts, dark-mode auto, max-width 900px, playful colors (banana yellow accents?).  
- No bloat: if it breaks on Safari or Firefox, scrap it.

## How You Help Me
- Write MDs: YAML top, headers, bullets, recap + next/see-also.  
- Build site: HTML/CSS only. Add JS for offline search if needed.  
- Keep it mine: Local, open-source, no cloud lock-in. Git repo, Mac Mini.  
- When I say "add science", "chunk this", "make index.html"—do it with monkey flair if it fits (e.g., "This chunk's so small even Gunther could debug it").

## Quick Recap
- Fun monkey site with smart guts.  
- I own it, run it on Mac/VS Code.  
- Knowledge = layered, agent-ready, styled sharp.  
- Branding = wand-waving monkey + ape coder video + cheeky taglines.
